## 月结 LIS 上传按钮「点击无反应」根因更正（commit fd5bba9，已 push，部署中）
- 此前把按钮失效归因为「未选对仪器 / 浏览器缓存旧 JS」，但用户两次反馈，需更精确定位。
- 真因（代码层）：`QCList.vue` 上传按钮 `:disabled="!uploadInstrumentId"` —— 只要上方「质控仪器」下拉未选，按钮即灰显，点击被静默吞掉（无任何提示），正是「点击无反应」；即便已选仪器按钮可用，`triggerCsv` 内 `csvInput.value?.click()` 若 ref 为 null 也会因可选链变成静默 no-op。
- **更深层真因（浏览器自动化验证后发现）**：`<input ref="csvInput">` 位于 `el-tab-pane` 的 `v-for` 内部，Vue 3 会把该模板 ref 绑定成 **Proxy(Array)**（即使只渲染一个元素），所以 `csvInput.value.click` 是 `undefined`，导致「点了没反应」。
- 修复（commit 1608f14，已 push+部署）：
  1. 去掉 `:disabled`，按钮始终可点；未选仪器时 `ElMessage.warning` 提示。
  2. `triggerCsv` 内用 `Array.isArray(csvInput.value) ? csvInput.value[0] : csvInput.value` 解出真实 DOM 元素再 `.click()`。
- 验证：本地 dev server（代理到线上后端）+ Playwright 自动化登录 admin → QC 月结页 → 未选仪器点击按钮出现提示；选择罗氏 e601 后点击按钮成功触发 `filechooser` 事件。线上 chunk 已切为 `QCList-DOFwT-w7.js`，包含 `Array.isArray` 与提示字符串。
- 教训：模板 ref 放在 `v-for` 内部时，即使只渲染单个元素也会变成数组；Vue 3 文档有明确说明，但容易忽略。对关键按钮应先用浏览器自动化验证点击链路，不能仅靠代码审查。
