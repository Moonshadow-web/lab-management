## 项目质量要求匹配 bug 修复（降钙素原/降钙素误匹配「钙」）
- 根因：`quality_requirements.py` `_find_best` 与 `qc_service.py` `_lookup_qr_goal` 用朴素 `a in b / b in a` 包含匹配，单字「钙」误入「降钙素原/降钙素/骨钙素」，前缀「降钙素」误入「降钙素原」。
- 修复（commit 35828d4，已 push + 部署，build mark `qr-safe-contains-pct-ct-2026-07-20`）：
  - 新增 `quality_requirements_seed.contains_same_item` 安全包含匹配：仅当「非包含部分」全为修饰性成分（标本前缀 总/无机/血清/血浆/尿液… + 括号单位码）才视为同一项目；括号外 `/、` 才按段拆分（避免 μg/L 的 / 误拆）。项目库名均为中文，英文代码只在别名，故不依赖括号代码相等（避免 TG 甲状腺球蛋白↔甘油三酯 碰撞）。
  - 矩阵 `_find_best`（step2/5）与 QC 月结 `_lookup_qr_goal` 两条路径同步改用该 helper。
  - 补充卫健委 NCCL `血清降钙素原(PCT)` tea=靶值±30%或±0.03μg/L（取大值）；seed 幂等只新增 1 行。
- 线上验证（内部 host）：降钙素原 wst空/bj 10%±30%或±0.03μg/L/nccl 靶值±30%或±0.03μg/L（取大值）；降钙素 wst空/bj空(确认北京市无)/nccl 靶值±30%；骨钙素 不再匹配钙；无机磷 正确匹配 磷±10%。
- 回归：对 124 个真实 TestItem 名比对，21 项变化均为**移除错误匹配**（旧靠 钠/钙/铁/白蛋白 子串误命中），无正确匹配丢失；编译通过。
