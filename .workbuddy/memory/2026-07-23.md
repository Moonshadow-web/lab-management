- 修复（commit 165aa19，已 push+部署，build mark `attachment-img-compress-maxpacket-2026-07-23`）：
  1. `main.py` 启动时 `SET GLOBAL max_allowed_packet=134217728` 并 `engine.dispose()` 重建连接池（后端与 MySQL 同 VPC 可达）。
  2. 新增 `services/attachment_compress.py`：上传图片缩放(长边≤2400)+JPEG q85；依赖新增 Pillow。两个上传接口接入；超限返回 413 友好提示。
- 验证（线上端到端）：9MB 非解码字节两模块均 201（证明 max_allowed_packet 已提到 128MB）；535KB 真实照片上传后线上存储 363KB（压缩比 67.8%）、拉取 200。测试附件已清理。
- 遗留建议：SET GLOBAL 可能被托管实例重置 → 提醒用户在 TDSQL-C 控制台把 `max_allowed_packet` 永久设为 134217728，否则容器重启后大文件又 500。
- 控制台永久设置步骤（官方文档 product/1003/62740）：TDSQL-C 控制台→集群列表→点目标集群ID→「参数设置」页→搜 `max_allowed_packet`→「参数运行值」列点编辑输入 `134217728`(须为1024倍数，本值符合)→保存→弹窗选「立即执行」→确定。该参数是全局参数，须在「读写实例」(主实例)改，对集群所有实例生效；动态参数通常无需重启。验证：`SHOW GLOBAL VARIABLES LIKE 'max_allowed_packet';` 返回 134217728。
