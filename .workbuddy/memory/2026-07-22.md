## 附件预览 JPG 裂图修复
- 用户反馈仪器间/室间比对原始结果中 jpg 无法预览（裂图）。
- 根因：后端 `get_current_user` 仅读取 `Authorization: Bearer` 请求头，而 `<img>` / `<iframe>` / `window.open` 等浏览器原生请求无法携带自定义头，导致附件 URL 返回 401。
- 修复：
  - `backend/app/core/security.py`: `OAuth2PasswordBearer` 改为 `auto_error=False`，`get_current_user` 增加 `Request` 参数，按 **Authorization 头 → URL query token → cookie access_token** 顺序取 token。
  - `frontend/src/api/comparison.js` / `interlab.js`: `attachmentUrl` / `interlabAttachmentUrl` 在 URL 后附加 `&token=<access_token>`。
  - `backend/app/api/v1/diag.py`: build mark 更新为 `attachment-auth-token-2026-07-22`。
- commit `e522927`，已 push 到 origin/main，部署任务 `7ZQyHi` 后台执行中。
