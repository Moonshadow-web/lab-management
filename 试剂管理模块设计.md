# 试剂管理模块 · 设计规划

## 一、需求总览

| 功能 | 描述 | 频率 |
|------|------|------|
| 试剂目录 | 所有试剂/校准品/质控品/耗材，含品牌、规格、材料编码(设备处订购编码) | 基础数据，维护后基本不变 |
| 盘库（库存更新） | 记录试剂余量（批号/效期/数量） | 每月末必做 + 月中可按需 |
| 试剂订购 | 月初(10号前)大批量订购 → 导出向设备科提交的订单(编码+名称+数量)；可后续加订 | 每月初 + 不定期加订 |
| 试剂接收 | 到货录入：名称、数量、批号、效期、送货人、接收人 → 库存自动增加 | 每批到货 |
| 月消耗计算 | 月底盘库后系统自动算：月消耗 = 上期结余 + 本期入库 - 期末结余 | 每月末 |

## 二、数据模型（6张表）

### 1. reagent_items —— 试剂项目目录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| type | String(20) | 试剂/校准品/质控品/耗材 |
| category | String(50) | 大类：生化/免疫/凝血/血气等 |
| name | String(200) NOT NULL | 试剂名称 |
| brand | String(100) | 品牌（罗氏/雅培/迈瑞等） |
| spec | String(200) | 规格（如 400测试/盒）|
| material_code | String(50) | **设备处订购材料编码**（核心字段，用户提供Excel） |
| unit | String(20) | 基本单位（盒/瓶/支/人份） |
| manufacturer | String(100) | 生产厂家 |
| supplier | String(100) | 供应商 |
| min_stock | Integer | 最低库存预警量（用于订购参考） |
| is_active | Boolean | 启用/停用 |
| remark | Text | |
| created_at / updated_at | DateTime | |

> **上线第一步**：从用户提供的两个Excel导入到 `reagent_items`（`生化试剂订购代码.xls` + `器械科订试剂编码.xlsx`）

### 2. reagent_stock —— 库存余额（实时库存）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| item_id | FK → reagent_items.id | 关联试剂 |
| batch_no | String(100) | 批号 |
| expiry_date | Date | 效期 |
| quantity | Integer NOT NULL DEFAULT 0 | 当前库存数量 |
| last_updated | DateTime | 最后更新时间 |

> 每次接收自动增加，每次盘库更新，订购时不影响库存（订购后→接收时才增加）

### 3. reagent_inventory_check —— 盘库记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| check_date | Date NOT NULL | 盘库日期 |
| check_type | String(20) | 月末盘库/月中盘库 |
| operator | String(100) | 盘库人 |
| remark | Text | |
| created_at | DateTime | |

### 4. reagent_inventory_items —— 盘库细项

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| check_id | FK → reagent_inventory_check.id | 关联盘库 |
| item_id | FK → reagent_items.id | 关联试剂 |
| batch_no | String(100) | 批号 |
| expiry_date | Date | 效期 |
| recorded_quantity | Integer NOT NULL | 盘库记录的实际余量 |
| created_at | DateTime | |

> 盘库后更新 `reagent_stock` 余额

### 5. reagent_orders —— 订购单

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| order_no | String(50) UNIQUE | 订单号（自动生成：ORD-YYYYMM-XXX） |
| order_date | Date NOT NULL | 订购日期 |
| order_type | String(20) | 月初订购/加订 |
| status | String(20) | 草稿/已提交/部分到货/完成 |
| operator | String(100) | 操作人 |
| remark | Text | |
| created_at / updated_at | DateTime | |

### 6. reagent_order_items —— 订购细项

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| order_id | FK → reagent_orders.id | 关联订购单 |
| item_id | FK → reagent_items.id | 关联试剂 |
| ordered_quantity | Integer NOT NULL | 订购数量 |
| unit_price | Numeric(10,2) | 单价（可选填）|
| received_quantity | Integer DEFAULT 0 | 已到货数量 |
| remark | Text | |

### 7. reagent_receivings —— 到货接收

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| receipt_no | String(50) UNIQUE | 收货单号（自动生成：RCV-YYYYMM-XXX） |
| receipt_date | Date NOT NULL | 收货日期 |
| order_id | FK → reagent_orders.id NULLABLE | 关联订购单（可选，加订可无关联） |
| delivery_person | String(100) | 送货人 |
| receiver | String(100) NOT NULL | 接收人 |
| remark | Text | |
| created_at | DateTime | |

### 8. reagent_receiving_items —— 接收细项

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| receiving_id | FK → reagent_receivings.id | 关联收货单 |
| item_id | FK → reagent_items.id | 关联试剂 |
| batch_no | String(100) | 批号 |
| expiry_date | Date | 效期 |
| quantity | Integer NOT NULL | 到货数量 |
| remark | Text | |

**关键业务逻辑**：到货接收后自动更新 `reagent_stock`（按 item_id+batch_no 合并或新增行）。

### 9. reagent_consumption —— 月消耗记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| item_id | FK → reagent_items.id | 关联试剂 |
| year_month | String(7) | 年月（如 2026-07）|
| opening_balance | Integer | 月初库存（上月末结余） |
| total_received | Integer | 本月入库总数 |
| closing_balance | Integer | 月末库存（本次盘库） |
| consumption | Integer | **月消耗 = opening + received - closing** |
| calculated_at | DateTime | 计算时间 |

## 三、后端 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| **试剂目录** | | |
| GET | /api/v1/reagent-items | 列表查询（分页，可按类型/名称/品牌筛选）|
| POST | /api/v1/reagent-items | 新增试剂 |

| DELETE | /api/v1/reagent-items/{id} | 删除试剂（仅admin）|

| **库存** | | |
| GET | /api/v1/reagent-stock | 当前库存列表（实时余额） |
| GET | /api/v1/reagent-stock/{item_id} | 某试剂的库存明细（按批号） |
| **盘库** | | |
| GET | /api/v1/inventory-checks | 盘库记录列表 |
| POST | /api/v1/inventory-checks | 新建盘库（提交盘库细项后自动更新stock） |
| GET | /api/v1/inventory-checks/{id} | 盘库明细 |
| **订购** | | |
| GET | /api/v1/reagent-orders | 订购记录列表 |
| POST | /api/v1/reagent-orders | 新建订购单（含细项） |
| PUT | /api/v1/reagent-orders/{id} | 编辑订购单（草稿状态可改） |
| DELETE | /api/v1/reagent-orders/{id} | 删除订购单 |
| GET | /api/v1/reagent-orders/_export-order/{id} | 导出向设备科提交的订购表（Excel，含材料编码+名称+数量） |
| GET | /api/v1/reagent-orders/_export-fill-form/{id} | 导出打印版空白填写表 |
| **到货接收** | | |
| GET | /api/v1/reagent-receivings | 接收记录列表 |
| POST | /api/v1/reagent-receivings | 新建收货（自动增加stock） |
| GET | /api/v1/reagent-receivings/{id} | 收货明细 |
| **月消耗** | | |
| GET | /api/v1/reagent-consumption | 月消耗查询（可按年月筛选） |
| POST | /api/v1/reagent-consumption/_calculate | 手动触发月消耗计算（盘库后） |

## 四、前端页面规划

| 路由 | 页面 | 功能 |
|------|------|------|
| /reagent/items | 试剂目录 | 表格展示所有试剂/校准品/质控品/耗材，搜索筛选，导入Excel，编辑 |
| /reagent/stock | 实时库存 | 显示当前库存余额（含批号/效期/数量），库存不足预警 |
| /reagent/inventory | 盘库管理 | 盘库记录列表，新建盘库（弹窗录入各试剂余量），自动更新stock |
| /reagent/orders | 订购管理 | 订购单列表+状态，新建订购（选择试剂+填数量），导出订购表 |
| /reagent/receivings | 到货接收 | 收货记录列表，新建收货（关联订单或自由录入），自动入库存 |
| /reagent/consumption | 月消耗统计 | 按月查询各试剂消耗量，导出报表 |

**左侧菜单**：在菜单中加"试剂管理"菜单项，展开子菜单（目录/库存/盘库/订购/接收/消耗）。

                    └─────────┬──────────┘
                              │
                    ┌─────────▼──────────┐
                    │  月中按需盘库        │
                    │  更新实时库存        │

                    ┌─────────▼──────────┐
                    │  月末盘库            │

## 七、技术要点

- **Excel导入/导出**：后端用 `openpyxl`（已通过 pypdf 验证系统 Python 可装，备选 `xlrd` 读 `.xls`）；前端导出用 `exceljs`（前端 package.json 已装 `^4.4.0`）
- **订购表导出**：向后端发请求，后端生成 `.xlsx` 文件返回，前端下载
- **盘库操作**：弹窗中列出所有 active 试剂，逐行填写余量（可批量填0表示用完），提交后后端批量更新 `reagent_stock` 并写入 `reagent_inventory_items`
- **库存安全**：`reagent_stock` 每次更新走事务（SQLite 不支持并发高，但在线只有一个用户操作，OK）
- **权限**：管理员 + lab_technician 可操作，普通用户只读
- **菜单**：在侧边栏"试剂管理"父菜单下，子菜单为：目录、库存、盘库、订购、接收、消耗
