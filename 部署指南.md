# 上线部署指南

> 生免组实验室管理系统 → 腾讯云开发 CloudBase Run

## 前置条件

1. **腾讯云账号**（已有）
2. **开通 CloudBase**：访问 https://console.cloud.tencent.com/tcb → 创建环境（选"按量计费"，有免费额度）
3. **安装 CloudBase CLI**：`npm install -g @cloudbase/cli`
4. **登录 CloudBase**：`tcb login`

---

## 部署步骤

### 第 1 步：生成生产密钥

在项目根目录执行，生成一个随机密钥替换 `cloudbaserc.json` 里的 `SECRET_KEY`：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

把输出填入 `cloudbaserc.json` 的 `envs.SECRET_KEY`。

### 第 2 步：填入环境 ID

在 CloudBase 控制台创建环境后，把环境 ID 填入 `cloudbaserc.json` 的 `envId` 字段。

### 第 3 步：部署到 CloudBase Run

```bash
cd /d/workbuddyprojects/网页版-生免速查工具
tcb run deploy --config cloudbaserc.json
```

部署完成后，CloudBase 会分配一个域名，如：
`https://lab-management-xxxxx.tcloudbaseapp.com`

把这个域名填回 `cloudbaserc.json` 的 `envs.FRONTEND_ORIGIN`，然后重新部署一次。

### 第 4 步：迁移现有数据（关键）

新部署的容器 `data/` 目录是空的，需要把本地已有的数据库和上传文件迁移过去。

**方法 A：通过 API 导入（推荐）**
1. 访问 `https://你的域名`，用 `jinzizheng / 123456` 登录
2. 改密后，在「项目查询」页用「批量导入」导入项目数据
3. 其他模块手动补录

**方法 B：通过 CloudBase CLI 上传文件到持久卷**
```bash
# 上传数据库
tcb run cp data/app.db lab-management:/app/data/app.db

# 上传上传文件目录
tcb run cp -r data/uploads/ lab-management:/app/data/uploads/
```

### 第 5 步：验证

1. 访问 `https://你的域名/health` → 应返回 `{"status":"ok"}`
2. 访问 `https://你的域名` → 应显示登录页
3. 用 `jinzizheng / 123456` 登录 → 首次改密 → 进入系统
4. 检查各模块数据是否正常

## 数据库说明

- **当前方案**：SQLite（单文件 `app.db`，1.2MB），存在持久卷 `/app/data/` 中
- **适用场景**：19 个用户、并发写少，SQLite 完全够用
- **后续升级**：如需迁移到 CloudBase MySQL，只需：
  1. 在 CloudBase 控制台创建 MySQL 实例
  2. 设环境变量 `DATABASE_URL=mysql://用户名:密码@主机:3306/数据库名`
  3. 运行迁移脚本导出 SQLite 数据 → 导入 MySQL
  4. 重新部署

## 文件存储说明

- **当前方案**：本地磁盘（持久卷 `/app/data/uploads/`，89MB）
- **后续升级**：可切到 CloudBase 对象存储，设 `STORAGE_BACKEND=cloud`

## 备份

- **数据库**：定期 `tcb run cp lab-management:/app/data/app.db ./backup/app-$(date +%Y%m%d).db`
- **上传文件**：定期 `tcb run cp -r lab-management:/app/data/uploads/ ./backup/uploads/`
- **建议**：设一个定时任务（自动化），每天备份一次

## 环境变量速查

| 变量 | 说明 | 示例 |
|---|---|---|
| `SECRET_KEY` | JWT 签名密钥（生产必改） | 随机 64 字符 |
| `FRONTEND_ORIGIN` | 前端域名（CORS） | https://xxx.tcloudbaseapp.com |
| `DATABASE_URL` | 数据库连接串（默认 SQLite） | mysql://user:pass@host:3306/db |
| `STORAGE_BACKEND` | 存储后端（local/cloud） | local |
| `SMTP_HOST` | 邮件服务器 | smtp.qq.com |
| `SMTP_USER` | 邮件账号 | 815268425@qq.com |
| `SMTP_PASS` | 邮件授权码 | 你的授权码 |

## 备选方案：腾讯云轻量服务器（更简单）

如果 CloudBase Run 配置复杂，可以先用轻量服务器：

1. 购买腾讯云轻量服务器（2核2G，约 50 元/月）
2. SSH 登录，安装 Python 3.13 + Node 20 + nginx
3. 上传项目代码
4. `cd backend && pip install -r requirements.txt`
5. `cd frontend && npm install && npm run build`
6. 用 nginx 反代：前端静态文件 + `/api` 转发到 uvicorn 8123
7. 配置 HTTPS（免费证书）

这种方式与本地开发完全一致，零代码改动，适合快速上线。
